# EAS Build Configuration

## Git Configuration

EAS Build requires a properly configured git repository. If you encounter git-related issues:

### Option 1: Use the build script (Recommended)
```bash
npm run build:android
# or for production
npm run build:android:production
```

This script automatically handles git issues and falls back to `EAS_NO_VCS=1` if needed.

### Option 2: Manual build with git fallback
```bash
EAS_NO_VCS=1 npx eas-cli build --platform android
```

### Option 3: Fix git repository
If you want to use git properly:

1. Ensure you have a remote configured:
   ```bash
   git remote -v
   ```

2. If no remote exists, add one:
   ```bash
   git remote add origin <your-repo-url>
   ```

3. Ensure all changes are committed:
   ```bash
   git add .
   git commit -m "Your commit message"
   ```

## Troubleshooting

### Git clone errors
If you see `git clone` errors, use `EAS_NO_VCS=1` flag or the build script.

### Missing blobs
If you see missing blob errors, run:
```bash
git gc --prune=now
```

### Version code conflicts
EAS automatically increments versionCode. If you need to set it manually, edit `app.json`:
```json
{
  "expo": {
    "android": {
      "versionCode": 32
    }
  }
}
```
